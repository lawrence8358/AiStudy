# AI 軟體工程

## 🧭 建立 SubAgent
> 情境：從零建立多代理調度中心，含平行處理、重試流程與結案報告設計

``` markdown
我要建立一個 API 開發的調度中心，請先使用子代理了解一下調度中心參考範本.md，接著用另外一個子代理了解一下 API 後端開發結構，我需要你協助我完成底下的需求
1. 建立 pex-dev-api-commander.agent.md 的 API 調度中心，其中多個子代理的作業流程為 [分析 → 單元測試框架 → 開發 → 建置與單元測試 → 程式碼審核 → 結案分析報告]
   + 若建置與單元測試未通過，請通知調度中心重新指派開發完成修正。
   + 若程式碼審核未通過，請重新通知調度中心重新指派 開發 → 建置與單元測試。
2. 若經分析可平行處理項目，也記得參考在 subagent 內標示。
3. 各專責 Subagent 請開啟所需的 tool 即可
4. 若各專責 Subagent 需要 Skill 也可以建立所需的 Skill
5. 若有重試流程預設請指定 5 次，若超過次數也不要自行停止作業，而是詢問我如何處理，再依照我的指示決定後續動作。
6. 完成後請新增一個 md 檔案描述各檔案的目的，路徑等相關辨別資訊
```
