# OpenSpec：需求調查與執行開發

## 呼叫 OpenSpec 進行需求調查(1)
> 情境：新增台南地區 API 支援、合併雙縣市回應，並完成 Android 整合測試

``` markdown
/opsx:new 請先調查了解一下 @README.md  和 @FrontendV2/README.md  內的功能，同時並了解一下 @FrontendV2/ 專案 。

原本這個 APP 只有支援高雄地區 Header 為 ['Referer': 'https://kepbgps.kcg.gov.tw'] 的三個 API
1. https://kepbgps.kcg.gov.tw/WebService/WsSkyeyes.asmx/Newgetlocation
2. https://kepbgps.kcg.gov.tw/WebService/WsSkyeyes.asmx/NewgetCarsinfo
3. https://kepbgps.kcg.gov.tw/WebService/WsSkyeyes.asmx/getcarline

我目前希望可以加上台南地區的 Header 為 ['Referer': 'https://clean.tnepb.gov.tw'] 的三個 API
1. https://clean.tnepb.gov.tw/WebService/WsSkyeyes.asmx/Newgetlocation
2. https://clean.tnepb.gov.tw/WebService/WsSkyeyes.asmx/NewgetCarsinfo
3. https://clean.tnepb.gov.tw/WebService/WsSkyeyes.asmx/getcarline

同時在發出 API 請求的時候並須等待高雄和台南兩個 API 合併後再一起回傳，但必須考慮若其中一個縣市的 API 異常時(可能超時或壞掉)不能影響另外一個縣市。

請在開發時請注意重用性，避免不好維護的問題，並通過前端建置。

並且在完成後請記得使用 Android 開啟並使用 Livemode 完成整合測試，再通知我
最後若有需要也記得更新兩份 Readme 文件
```

## 呼叫 OpenSpec 進行需求調查(2)
> 情境：CLI 新增 ConnectionString 參數、WPF 新增自動建立資料庫設定，含單元測試

``` markdown
/opsx:new 請先調查了解一下 README.md 內的功能，這個專案共有 CLI & WFP 兩個專案，我有底下的需求需要你協助調整

1. 原本 CLI 是透過多個參數來指定資料庫相關設定(eg. host & port & db name 等等)，但如此一來，我想要整合 CI 就必須使用多個變數來設定，我希望可以在 CLI 那邊增加一個 ConnectionString 的方式來指定。
2. WPF & CLI 都必須都在有資料庫的前提下，才能使用此功能，我希望 WPF 多一個設定畫面 & CI 則是多一個參數設定，當傳入該設定時，預設會檢查若資料庫不存在，則自動建立，但這個情境可能需要考量測試連線時資料庫不存在(都有勾選時，僅測試不指定資料庫的連線狀況) & 比對資料表不存在(等同所有資料表都是新增)等狀況處理，當有勾選時。
3. 請先規劃好單元測試後再開始進行功能開發，最後再進行單元測試。
4. 完成後請記得回頭更新 README.md & README.zh-TW.md 兩份規格文件。 
```
