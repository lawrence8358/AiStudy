# 參考樣本如下 : 
``` markdown
工作項目主題 : 加值模組套件化 POC

### 1. 故事說明
為優化各專案維護效率，依 3/16 會議決議使用 Log 當作加值模組 POC 的範本。目標是將 Log 功能轉變為獨立的 NuGet 套件 與 前端獨立目錄，確保未來各專案可透過版本更新快速修復 Log 相關問題，且在不需此功能時能快速移除而不影響系統執行。
 
---
 
### 2. 需求說明
A. 後端部分
+ 專案拆分：參考 Hangfire 設計，建立公司內部套件獨立專案。
+ 禁止相依：嚴禁回頭參考共用版本程式。
+ DI 注入機制：
    + 例如，提供 services.AddCustomLog(options => { ... }) 擴充方法。
    + 需可動態傳入 DB Connection String 及其餘擴展設定（如：Retention Days）。
+ 遷移內容：包含原本底層的 Core First 機制、Entity、Model、Service 與 Controller。
+ 單元測試：請務必將重要功能補上單元測試。

B. 套件化與 CI/CD
+ 版本號碼：遵循 Semantic Versioning (語意化版本)，但原則上內部套件非必要，不應出現主版號變更，若有需提供 Breaking change 文件。
+ 推送目標：公司內部私有 NuGet Server。
+ 自動化：完成 CI/CD 設定，確保程式碼合併後自動產出新版套件。

C. 前端部分
+ 搬遷路徑：將所有 Log 相關檔案移至 src/package/log。
+ 檔案範圍：包含 LogView.vue、LogService.ts、LogModel.ts 等。
+ 結構要求：確保模組具有獨立性，目的是當刪除 src/package/log 目錄時，主程式+ 不需修改代碼即可正常建置。
 
---
 
### 3. 測試情境
+ DI 配置測試：在 Program.cs(亦可將設定透過擴展的方式集中在特定資料夾內) 注入不同的資料庫連線字串，
+ 拔除測試：手動移除後端 NuGet 參考及前端 src/package/log 資料夾，不應影響原公版程式運作。
+ 版本更新測試：發布新版 Log 套件至 NuGet 倉庫，並在現有專案執行更新後，必須套用新增功能。
```
