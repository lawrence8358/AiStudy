# 資料庫

## 📊 Postgres 佈署工具
> 情境：執行單元/整合測試、敏感資料審查並移至 .env，最後產生測試報告

``` markdown 
1. 使用兩個子代理查看 #file:test  並分別進行單元測試和整合測試，務必要通過驗證，若無法通過請修正相關程式或腳本直到通過為止，我的 pstgreSQL 是使用 docker 安裝的，PG 相關連線資訊如下。
"""
DB_HOST="localhost"
DB_PORT="5432"
DB_NAME="your_db_name"
DB_USER="your_db_user"
DB_PASS='your_db_password'
"""

2. 上述單元測試和整合測試通過後，請再用另外一個子代理，針對單元測試或整合測試檢查是否包含敏感數據，若有請改用 .env 替代(先改到那邊還是使用敏感資訊)，並在每個變數加上註解說明後，重新執行單元和整合測試，並在結束後產生一份測試報告，內容包含通過 & 未通過的功能清單。

3. 在後再使用另外一個子代理進行整個內容是否包含敏感資訊的檢查，除了 .env 外，其餘若包含敏感資訊，請修正，並將最終結果回報給我。

還是請記住，你只需要指揮並回報，任何調整和修改都請子代理協助處理
```
