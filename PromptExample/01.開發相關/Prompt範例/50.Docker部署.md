# Docker 部署

## Docker 環境定
> 情境：設定容器對外 port、健康檢查驗證，並將連線資訊從檔案移至環境變數

``` markdown 
這是一個 NET Core 的站台請使用子代理調查一下 PrimeEagleX\AP\DockerFile 檔案和其建置設定，目前的資料庫連線資訊是由 PrimeEagleX\AP\Backend\appsettings.Secrets.json 這個檔案設定的。

我需要你協助我將站台設定在容器內使用 5000 port 啟動，並且讓我的站台在本地端可以透過 5005 連線到。

在完成設定後可能要一段時間等候容器啟動，因此請呼叫另外一個子代理 http://localhost/api/healthcheck 驗證是否成功，若不成功請呼叫建置容器的代理修正功能。

最後當站台測試完畢後，我希望你呼叫另外一個代理，把 appsettings.Secrets.json 的連線資訊改到 docker-compose.yaml 的環境變數，但這邊你可能要確保先把原本的設定調整成另外一個名稱，以確保修正是有效的，完成後一樣要通過驗證。
```

## Docker 設定
> 情境：將 PostgresSQL 佈署工具整合進 Dockerfile，資料庫佈署完成後再啟動站台

``` markdown 
另外我有用 NET Core 10 寫了一個 PostgresSQL 佈署工具，並放在 https://github.com/lawrence8358/PostgresDeployer/releases/download/v1.0.2/cli.zip 這個連結。

我希望你能幫我把這個工具打包到，PrimeEagleX\AP\DockerFile 的流程內，並協助解壓縮後刪除下載的壓縮檔案，請寫成一行以最小化 Image Layer，這裡面會有一個工具 pgdeploy

接著在 Copy 完後端的程式後，記得把 PrimeEagleX\Publish\DbScript 內的檔案複製進去，接著底下這個是執行的範例，並在完成資料庫佈署後，再啟動站台

pgdeploy deploy ^
  --host "database host" ^
  --port 5432 ^
  --database "PrimeEagleX" ^
  --username "your_db_user" ^
  --password "password" ^
  --schema "Db\Public" ^
  --init-data "Db\InitalizeScripts" ^
  --extensions "pgcrypto,postgis" ^
  --yes

資料庫佈署的 CLI 使用方式也可以到這邊調查 https://github.com/lawrence8358/PostgresDeployer。

別忘了，你只是指揮，千萬不要動手，請開始吧。
```

## 超級調度中心
> 情境：整合 Redis、RabbitMQ、調整基底映像，將所有敏感設定移至 .env 並驗證站台

``` markdown
1. 我的容器的基底是 NET 10 `mcr.microsoft.com/dotnet/aspnet:10.0`，資料庫佈署工具是 NET 10，但因為我的應用程式站台是 NET 8，所以 DockerFile 的腳本內除了使用基底外，又額外了下載 dotnet 8，但因為這並非我本意，因為我最終的執行環境還是希望依站台為主(避免誤會)，資料庫佈署工具只是過渡時期，因此請幫我把基底改成 dotnet 8，然後再下載資料庫佈署工具時，一併下載所需的 NET 10 版本(一樣使用單一 Layer)

2. 資料庫下載工具新增了 --connection-string 和 --create-db-if-not-exists 兩個屬性，請參考 https://github.com/lawrence8358/PostgresDeployer 內的說明，我要將原本的多個資料庫參數改成統一使用 Docker-Compose.yaml 內的連線字串就好(這樣就不用重複定義)，並將這些環境變數的敏感資料移動到 .env 內(須提供各設定的註解)。

3. 我的系統本來有使用 Redis 設定，同時也有設定底下的設定，請幫我把這個也加到 docker-compose 中，並給予帳號密碼(一樣定義到環境變數中)
"""
save 1 1
# save 900 1
save 300 10
save 60 10000
"""

4. 我的系統也有使用 RabbitMQ，請幫我把這個也加到 docker-compose 中，一樣若有相關帳密(一樣定義到環境變數中)。

5. 我的站台原本有一個 PrimeEagleX\Publish\Backend\appsettings.Secrets.json.disabled，這裡面包含了所有可變動參數，包含上述提到的服務(若新增的 Redis 和 RabbitMQ 的帳號密碼已變更也請調整)，請全部幫我改成環境變數設定，並啟動 docker-compose，驗證 http://localhost:5005/api/healthcheck，直到回應成功為止，否則回報主代理要求其它子代理進行修正。


還是請記住，你只需要指揮並回報，任何調整和修改都請子代理協助處理，千萬不要動手，請開始吧。
```
